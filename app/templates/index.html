📖 API Documentation

      

        
All /api/* endpoints require an API key. Pass it via Authorization: Bearer <key> or X-API-Key: <key>.


        
Authentication

        

          

            POST/api/loginDashboard login
          

          

            
Authenticate to the dashboard UI. Sets a session cookie.

            
curl -X POST http://localhost:5001/api/login \
  -H "Content-Type: application/json" \
  -d '{"password": "admin"}'

          

        


        
Customers

        

          

            GET/api/customersList all customers
          

          

            
Returns all customers with order counts. Optional ?search=term filter. Email is the primary key.

            
curl http://localhost:5001/api/customers \
  -H "Authorization: Bearer ***"

          

        

        

          

            GET/api/customers/:emailGet a customer by email
          

          

            
curl http://localhost:5001/api/customers/john@gmail.com \
  -H "Authorization: Bearer ***"

          

        

        

          

            POST/api/customersCreate a customer
          

          

            
curl -X POST http://localhost:5001/api/customers \
  -H "Authorization: Bearer ***" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "jane@example.com",
    "firstname": "Jane",
    "lastname": "Smith",
    "username": "janes",
    "phone": "555-0100",
    "city": "Portland",
    "street": "Oak Ave",
    "number": 42,
    "zipcode": "97201"
  }'

          

        

        

          

            PUT/api/customers/:emailUpdate a customer
          

          

            
curl -X PUT http://localhost:5001/api/customers/jane@example.com \
  -H "Authorization: Bearer ***" \
  -H "Content-Type: application/json" \
  -d '{"city": "Seattle"}'

          

        

        

          

            DELETE/api/customers/:emailDelete a customer (and their orders)
          

          

            
curl -X DELETE http://localhost:5001/api/customers/jane@example.com \
  -H "Authorization: Bearer ***"

          

        

        

          

            GET/api/customers/:email/ordersGet customer's orders
          

          

            
curl http://localhost:5001/api/customers/john@gmail.com/orders \
  -H "Authorization: Bearer ***"

📄 Technical Documentation

      


        
Architecture Overview

        
Flask backend + single-page HTML/CSS/JS frontend. Dual database support (SQLite local, PostgreSQL production). API key auth for external access, session auth for the dashboard.


        

          
Browser (index.html)
  └─ apiFetch() + session cookie
       │
       ▼
Flask Server (app/server.py)
  ├─ CORS + Sessions middleware
  ├─ before_request → require_api_key()
  ├─ Route handlers (customers, orders, products, keys)
  └─ Database Abstraction Layer
       ├─ db_fetchone / db_fetchall / db_execute
       └─ Auto-translates ? → %s for Postgres
            │
            ▼
   ┌─────────────┐     ┌──────────────┐
   │   SQLite     │     │  PostgreSQL   │
   │ (local dev)  │     │  (Render)     │
   └─────────────┘     └──────────────┘

        


        
Data Model

        
Five tables with foreign key relationships:

        

          
TablePrimary KeyNotes

          
            
customersemail (string)No integer ID — email is the PK

            
ordersorder_number (5-digit int, starts 10001)FK → customers.email

            
order_itemsid (auto-increment)FK → orders.order_number, products.id

            
productsid (integer)Read-only — seeded from Fake Store API

            
api_keyskey_hash (SHA-256)Stores hashed keys; prefix for identification

          
        


        

          
customers (email PK)
    │
    ├── 1:N ── orders (order_number PK)
    │              │
    │              └── 1:N ── order_items
    │                              └── FK → products.id
    │
    └── api_keys (key_hash PK)

Cascade: DELETE customer → deletes their orders + items
Cascade: DELETE order → deletes its items

        


        
Authentication

        
Two auth mechanisms, both enforced via before_request:


        

          
1. API Key Auth — for external API calls

          
Three header formats accepted:

          
Authorization: Bearer fsk_xxx...
Authorization: ApiKey fsk_xxx...
X-API-Key: fsk_xxx...

          
Keys are SHA-256 hashed before storage. Master key auto-generated on first run. Create additional keys via POST /api/keys.

        


        

          
2. Dashboard Session Auth — for browser access

          
Login via POST /api/login sets a Flask session cookie. Dashboard-authenticated requests skip API key requirement. Password set via DASHBOARD_PASSWORD env var (default: admin).

        


        
Exempt Endpoints (No Auth Required)

Seeding & Data Generation

        
On first startup, if the products table is empty:

        
    
          
  1. Live API — Fetches products, users (→ customers), and carts (→ orders) from https://fakestoreapi.com
    2. 
          
  2. Fallback — If the API is down, uses embedded arrays (20 products, 10 customers, 7 orders)
    3. 
        

        
Batch endpoints use Faker to generate realistic random data: names, emails, addresses, and order line items with random products and quantities.


        
Schema Migration

        
On startup, _needs_migration() checks for the old schema (integer id column in customers). If detected, all tables are dropped and recreated with the new schema (email-based PK), then re-seeded.
